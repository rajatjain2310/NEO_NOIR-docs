top_n_combinations(sku, fr1, strategy, split_date, of_per_week, period, avg_leadtime, lead_time_var, demand_path, in_transit, in_stock, ordering_freq, budget_constraint, a_choice, b_choice, rop_choice, max_evaluations, price, opt_mode, holding_constraint, tolerance, n, w_hc=0.5, w_fr=0.5)¶
Parameters

  • sku (int) – Product entity

  • fr1 (float) – Fill rate threshold

  • strategy (str) – Demand Prediction Algorithm to be used

  • split_date – Date from which inventory starts

  • of_per_week – Hard limit on number of inventory orders to be placed per week :type of_per_week: int

  • period (int) – The number of days to run the simulation for (int)

  • avg_leadtime (int) – Constant arrival time in days for any inventory order (int)

  • lead_time_var (int) – Variability in historical lead times (std metric) (float)

  • demand_path (str) – Path to where main python file is stored

  • in_stock (int) – Initial Inventory Definition

  • in_transit (int) – In transit Inventory

  • ordering_freq (int) – Supplier constraints on inventory ordering period

  • budget_constraint (float) – Inventory order placed on supplier always less than this constraint

  • a_choice (array) – Values for ‘a’ parameter in MSN calculation

  • b_choice – Values for ‘b’ parameter in MSN calculation

  • rop_choice (array) – Values for re-order points parameter in MSN calculation

  • max_evaluations (int) – Maximum number of evaluations while optimization

  • price (int) – Price of SKU

  • opt_mode (str) – Choice of optimization. Choices are ‘opt_fr’, ‘opt_hc’ and ‘custom’

  • tolerance (float) – Tolerance in fill rate while optimization generally ranges between 0.01-0.05

  • holding_constraint (int) – Constraint on holding cost while optimization

  • n (int) – Top n optimizations results to be shown

  • w_hc (float (0,1)) – Weight associated to holding cost in custom setting

  • w_fr (float (0,1)) – Weight associated to fill rate or lost cost in custom setting

Returns

Dataframe with top n combinations of parameters for inventory optimization
